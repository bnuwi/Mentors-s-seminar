## Задача 2. ##

Изменения, внесённые через терминал, действуют только в текущей сессии, так как каждая новая сессия создаёт собственное окружение. 
Чтобы сделать изменения постоянными, новую директорию нужно добавить в файл ~/.bashrc, который выполняется автоматически при запуске каждого нового терминала. 
После редактирования .bashrc изменения вступают в силу после выполнения команды source ~/.bashrc или перезапуска терминала. 
Наши изменения временные, поскольку скрипт выполняется в дочернем процессе Bash. То есть терминал — это один процесс Bash.
А когда запускается наш скрипт ./update_path.sh, создаётся новый процесс, внутри которого и выполняется скрипт.
Когда скрипт делает: export PATH, то он изменяет переменную PATH только внутри себя (т.е внутри дочернего процесса).
После завершения скрипта этот дочерний процесс закрывается, и соответсвенно все его переменные исчезают.
Родительский процесс (а именно терминал) ничего о них и не знает.

## Задача 5 ##

**Наблюдения:**

- После запуска скрипта фоновые процессы создавались внутри процесса самого скрипта, и команда jobs, выполненная внутри скрипта, показывала все три задачи как Running
- После завершения скрипта фоновые процессы скрипта завершались, поэтому команды fg и bg в этом же терминале уже не имели объектов для управления и выводили ошибку об отсутствии процессов

**Демонстрация управления процессами в интерактивном терминале:**

fg и bg процессы были запущены напрямую в терминале:

```bash
sleep 30 &
sleep 30 &
sleep 30 &
jobs
fg %1      # перевод первой задачи на передний план
Ctrl+Z     # приостановка задачи на переднем плане
bg %1      # перевод задачи обратно в фон
jobs       # проверка состояния процессов
```

**Результаты наблюдений:**

- Команда jobs отображает список фоновых задач текущего терминала с их статусом (Running, Stopped) и номерами задач
- Перевод задачи на передний план fg блокирует терминал до завершения процесса
- Приостановка задачи (Ctrl+Z) меняет её статус на Stopped
- А bg позволяет процессу продолжить выполнение без блокировки терминала


## Задача 7 ##

1. В терминале создан временный alias:

```bash
alias ll='ls -la'
```

`ll` выводит все файлы в текущей директории с подробной информацией

2. Постоянный alias:

```bash
echo "alias ll='ls -la'" >> ~/.bashrc
source ~/.bashrc
```
После перезагрузки терминала alias `ll` доступен в любой новой сессии

3. Автодополнение:

- Для команды `cd`:   
  - Вводим `cd` и начало какой-либо папки:

```bash
cd D
```

теперь нажимаем клавишу `Tab` и Bash автоматически дополняет путь, если он уникален, или показывает список вариантов, если их несколько
